Trabalhando com dados em armazenamento

Introdução

Neste tópico, iremos ter uma introdução sobre os tipos de armazenamento no Flutter, como o armazenamento local como banco de dados, como arquivos, entre outros.

  • Tipos de armazenamento de dados
    Cache:
    É usado para armazenar temporariamente dados que são frequentemente acessados e não precisam ser persistentes entre as sessões do aplicativo. O Flutter oferece suporte ao armazenamento em cache por meio de bibliotecas como o shared_preferences, permitindo que você armazene dados simples como pares de chave-valor
    Local Database:
    Para aplicativos que precisam de um armazenamento mais robusto e estruturado, é possível utilizar um banco de dados local. O Flutter possui bibliotecas populares, como o sqflite e o moor, que permitem a criação e a manipulação de bancos de dados SQLite no dispositivo
    File Storage:
    Quando se trata de armazenar arquivos, como imagens, áudios ou documentos, o Flutter oferece facilidades para manipulação de sistemas de arquivos locais. A biblioteca path_provider é frequentemente utilizada para obter o caminho do diretório de armazenamento do dispositivo e a biblioteca dart:io oferece recursos para trabalhar com arquivos e diretórios
    Outros:
    Além dessas opções, também é possível integrar o Flutter com serviços de armazenamento em nuvem, como Firebase Storage, Amazon S3 ou Google Cloud Storage, para armazenar dados de forma remota e acessível em vários dispositivos
  • Banco de dados local
    Introdução:
    ● No desenvolvimento de aplicativos Flutter, é comum a necessidade de armazenar dados localmente
    ● Uma das formas de fazer isso é utilizando um banco de dados local para armazenar e recuperar informações no dispositivo do usuário
    ● Existem várias opções disponíveis para realizar essa tarefa em Flutter, sendo as mais populares o SQLite e o Moor
    SQLite:
    ● Um banco de dados relacional embutido, que oferece uma solução leve e eficiente para armazenamento local
    ● Amplamente utilizado em aplicativos móveis e possui uma implementação nativa em Flutter por meio do pacote "sqflite"
    ● É possível criar tabelas, executar consultas SQL e realizar operações de inserção, atualização e exclusão de dado
    Moor:
    ● Um gerador de código que facilita a interação com bancos de dados SQLite em Flutter
    ● Ele permite definir as tabelas e consultas através de classes Dart, gerando o código necessário para acessar o banco de dados
    ● Com o Moor, você pode escrever consultas em uma sintaxe mais amigável, além de aproveitar recursos como verificação de tipos em tempo de compilação
